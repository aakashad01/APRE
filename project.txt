📘 PROJECT KNOWLEDGE PACK — FULL START-TO-END DESCRIPTION
1. Project Title

A Hybrid Quantum-Classical Penetration Testing Framework for Detecting and Profiling Web Attacks

(Framework codename: Q-APRE — Quantum Attacker Persona Reconstruction Engine)

2. Project Overview

This project develops a general, reusable cybersecurity framework that detects, analyzes, and profiles web attackers using:

Web honeypots (with IDOR & SSRF vulnerabilities)

Penetration-testing based attack observation

Full-stack behavioral logging

Feature extraction and attack pattern mining

Hybrid Quantum-Classical Machine Learning (QNN)

Adaptive cyber deception responses

Automated forensic report generation (HIP: Hacker Intelligence Package)

The framework is sector-agnostic, modular, and can be deployed in any industry (FinTech, healthcare, e-commerce, SaaS, government, etc.).

3. Problem Statement

Traditional IDS/WAF systems only perform attack detection, lacking:

Attacker personality reconstruction

Behavioral understanding

Adaptive deception

Multi-step correlation between IDOR, SSRF, etc.

Explainable intelligence reports

Quantum-enhanced feature representation

This project solves these gaps by combining penetration testing, honeypots, QNNs, and deception.

4. System Goals

Deploy realistic web honeypots with IDOR + SSRF vulnerabilities.

Attract attackers and log all behavior.

Extract behavioral, timing, mutation, and payload features.

Use Hybrid Quantum Neural Networks (QNNs) to classify attacker personas.

Apply adaptive deception techniques based on persona.

Generate a complete forensic Hacker Intelligence Package (HIP) per session.

Allow the framework to be redeployed across industries with minimal changes.

5. Architecture (High-Level Workflow)
Attacker → Honeypot → Reverse Proxy + WAF → Logging → Sessionization 
→ Feature Extraction → QNN-Based Persona Classifier → Adaptive Deception 
→ HIP Output Folder

6. Honeypot Layer

The honeypot is a vulnerable web application (DVWA or custom Flask app) modified to create:

IDOR vulnerability

Example endpoints:

/user/<id>

/file?id=<id>

/transaction/<id>

No access control is applied to allow IDOR exploitation.

SSRF vulnerability

Endpoint:

/fetch?url=<attacker_input>

Instead of making real outbound requests, the honeypot:

Detects internal targets

Returns fake cloud metadata

Returns fake internal service data

Logs all SSRF attempts

This approach keeps the system safe.

7. Traffic Control Layer
Nginx Reverse Proxy:

Handles all HTTP traffic

Logs full request metadata

Hides internal infrastructure

ModSecurity (optional):

Applies OWASP CRS rules

Creates audit logs

Detects automated scanners or malformed payloads

All logs forward to the collector.

8. Logging & Sessionization Engine

The collector:

Reads Nginx & WAF logs

Groups events into sessions by:

IP address

Cookies

Time window (e.g., 60 sec)

Creates folder:

session_<timestamp>/
    raw_requests.log
    raw_responses.log
    metadata.yaml


Each session represents one attacker or one automated tool run.

9. Feature Extraction Engine

The extractor analyzes each session and generates four key files:

1. parsed_features.json

General features:

Total requests

Timing patterns

Automation indicators

Payload entropy

Header anomalies

User-agent patterns

2. idor_patterns.json

IDOR-specific features:

Sequential/Random ID probing

Success/fail patterns

Parameter mutation style

Encoding/obfuscation usage

3. ssrf_patterns.json

SSRF-specific features:

Target hosts

Port probing

Protocols tested

Internal metadata probing behavior

4. sequence_timeline.json

Ordered behavioral steps of the attacker.

These become ML/QNN input features.

11. Hybrid Quantum-Classical Machine Learning
Preprocessing

StandardScaler

PCA → reduce features → 4 PC components (for 4 qubits)

Quantum Encoding

Angle embedding:

RY(PC1), RY(PC2), RY(PC3), RY(PC4)

Variational Quantum Circuit (VQC)

4 qubits

2–3 variational layers

RX/RZ rotations

CNOT entanglement

Measurement

Pauli-Z expectations → 4-dimensional quantum embedding.

Classical Classifier

XGBoost or small neural network

Outputs persona + confidence + embedding vector
